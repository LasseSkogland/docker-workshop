Docker Compose Concepts Used

  • Multiple Services: Web and database containers orchestrated together
  • Volumes: Database data persists with named volumes
  • Bind Mounts: Configuration files and HTML from the host
  • Networking: Services communicate via a custom bridge network
  • Dependencies: Web service depends on database service
  • Environment Variables: Used to configure database credentials

Try These Commands

# Start the services
docker compose up -d

# View logs
docker compose logs -f

# List running containers
docker compose ps

# Stop the services
docker compose down

# Stop and remove all data
docker compose down -v
